# Ibis Profiling: JSON Output Specification

This document defines the schema and structure of the `REPORT_DATA` JSON generated by `ibis-profiling`. The output is designed for high-performance React frontends and maintains functional parity with `ydata-profiling`.

## 1. Overview & Modes

The report generator supports two execution modes:
- **Full Reporting (Default):** Exhaustive analysis including duplicates, second-pass moments (skewness), correlations, and detailed missing value visualizations.
- **Minimal Reporting:** A high-speed path that skips computationally expensive operations, suitable for massive datasets (10M+ rows) or fast previews.

## 2. Top-Level Structure

| Key | Type | Description |
| :--- | :--- | :--- |
| `analysis` | `Object` | Metadata about the profiling session (title, timing). |
| `table` | `Object` | Global dataset metrics (row/column counts, memory usage). |
| `variables` | `Object` | Keyed by column name; contains per-variable stats and histograms. |
| `alerts` | `Array` | List of data quality warnings (Unique, Constant, Skewed, etc.). |
| `correlations` | `Object` | Correlation matrices (Pearson, Spearman). |
| `missing` | `Object` | Data completeness visualizations (Bar, Matrix, Heatmap). |
| `sample` | `Object` | Data previews (e.g., `head`). |
| `package` | `Object` | Versioning information (`ibis-profiling: 0.1.0`). |

## 3. Section Details

### 3.1. `analysis`
| Key | Type | Mode | Description |
| :--- | :--- | :--- | :--- |
| `title` | `String` | Both | The custom or default title of the report. |
| `date_start` | `ISO8601` | Both | Start time of profiling. |
| `date_end` | `ISO8601` | Both | End time of profiling. |
| `duration` | `Number` | Both | Total execution time in milliseconds. |

### 3.2. `table`
| Key | Type | Mode | Description |
| :--- | :--- | :--- | :--- |
| `n` | `Int` | Both | Total number of rows. |
| `n_var` | `Int` | Both | Total number of variables/columns. |
| `memory_size` | `Number` | Both | Total estimated memory footprint in bytes. |
| `record_size` | `Number` | Both | Average memory usage per row. |
| `n_cells_missing` | `Int` | Both | Total missing values across all cells. |
| `p_cells_missing` | `Float` | Both | Global missingness percentage (0.0 - 1.0). |
| `types` | `Object` | Both | Counts of detected types (e.g., `{"Numeric": 10, "Categorical": 5}`). |
| `n_distinct_rows` | `Int` | **Full** | Count of unique rows in the dataset. |
| `n_duplicates` | `Int` | **Full** | Number of exact duplicate rows. |
| `p_duplicates` | `Float` | **Full** | Percentage of duplicate rows. |

### 3.3. `variables`
Each variable object is keyed by its column name. Content varies by inferred type:

#### Common Metadata (All Types)
- `type`: `Numeric`, `Categorical`, `Boolean`, or `DateTime`.
- `hashable`: `Boolean` (True if column can be used for grouping/distinct counts).
- `n_missing`, `p_missing`: Raw count and percentage of `NULL` values.
- `n_distinct`, `p_distinct`: Raw count and percentage of unique values.
- `is_unique`: `Boolean` (True if `n_distinct == n`).

#### Numeric Metrics
| Metric | Mode | Description |
| :--- | :--- | :--- |
| `min`, `max`, `mean`, `std` | Both | Basic moments. |
| `5%`, `25%`, `50%`, `75%`, `95%` | Both | Distribution percentiles. |
| `variance`, `kurtosis` | Both | Variance and Excess Kurtosis. |
| `range`, `iqr`, `cv` | Both | Range, Interquartile Range, and Coefficient of Variation. |
| `n_zeros`, `p_zeros` | Both | Zero-value statistics. |
| `n_negative`, `p_negative`| Both | Negative-value statistics. |
| `n_infinite`, `p_infinite`| Both | Count and percentage of infinite values. |
| `skewness`, `mad` | **Full** | Skewness and Median Absolute Deviation. |
| `monotonic_increasing` | **Full** | `Boolean` (True if non-decreasing). |
| `monotonic_decreasing` | **Full** | `Boolean` (True if non-increasing). |
| `histogram` | Both | Binned distribution (see Section 4). |
| `extreme_values_smallest`| Both | List of 5 smallest non-null values. |
| `extreme_values_largest` | Both | List of 5 largest non-null values. |

#### Categorical Metrics
| Metric | Mode | Description |
| :--- | :--- | :--- |
| `mean_length` | Both | Average character length of strings. |
| `min_length`, `max_length` | Both | Extremes of string lengths. |
| `histogram` | Both | Top 20 most frequent values (Top Values). |
| `length_histogram` | Both | Distribution of character lengths. |

### 3.4. `correlations` (**Full Only**)
Contains matrices (e.g., `pearson`, `spearman`) formatted as a list of dictionaries for tabular rendering:
```json
"pearson": [
  { "age": 1.0, "salary": 0.82, "tenure": 0.45 },
  { "age": 0.82, "salary": 1.0, "tenure": 0.38 },
  ...
]
```

### 3.5. `missing` (**Full Only**)
- `bar`: `{ "col1": count, "col2": count }` - Total missing counts per column.
- `matrix`: `{ "columns": [...], "matrix": [[bool]] }` - Boolean nullity matrix (sampled).
- `heatmap`: Pearson correlation of nullity between columns.

### 3.6. `alerts`
List of JSON objects representing data quality issues.

| Alert Type | Logic / Threshold | Level |
| :--- | :--- | :--- |
| `CONSTANT` | `n_distinct == 1` | `warning` |
| `UNIQUE` | `n_distinct == n` | `warning` |
| `HIGH_CARDINALITY`| `p_distinct > 0.5` (Categorical only) | `warning` |
| `MISSING` | `p_missing > 0.05` | `info` |
| `ZEROS` | `p_zeros > 0.10` | `info` |
| `SKEWED` | `abs(skewness) > 10` | `info` |

#### Alert Object Structure
- `alert_type`: `String` (from table above).
- `fields`: `Array` containing the affected column name(s).
- `level`: `warning` (High) or `info` (Normal).
- `value`: (Optional) The metric that triggered the alert.

## 4. Visualization Schemas

### 4.1. `histogram` (Categorical/Numeric)
- **Categorical:** `bins` contains labels (string values); `counts` contains raw frequency.
- **Numeric:** `bins` contains range strings (e.g., `"[0.0, 10.0]"`); `counts` contains binned frequency.

## 5. Serialization Rules
1. **Strict JSON:** `NaN` and `Inf` are converted to `null`.
2. **Temporal:** `datetime` and `date` are converted to ISO 8601 strings.
3. **Ibis Scalars:** All deferred expressions are executed to primitives (int/float/bool/str) before export.
